Excel VBA关闭文件遇到错误的解决方法

使用VBA关闭打开的工作簿时候遇到问题。不能关闭工作簿。遇到这个错误的时候，可能需要有这几个方向想。

  1. 使用 application.displayalert = false ，这样是不显示警告信息，例如是否保存工作簿。
  2. 可以使用workbookobjectname.saved = true来让Excel强制认为workbookobject这个工作表已经保存。
  3. workbookobjectname.activate先将需要关闭的活动工作簿设为活动工作簿。
  4. 关闭的时候 workbookobjectname.close(0),其中的0也是savechanges:=false

上面的workbookobjectname是你的工作簿对象的名称。

一般来说，第四种方法是最直接有用的，但是，有的时候，却还是关不了，试验了很多次后，发现使用配合使用第3个方法非常有效的。

Carousel - 你生命所有重要瞬间，在云端

前面也注册了Loom，Loom是一个基于云的图片存储服务，然后早几天收到Loom的邮件，说Loom加入了Dropbox，而在前面Dropbox也发布了Carousel，可以推测是Dropbox想做一个单独的相册工具，于是收购了Loom，在此之前Dropbox已经有Camera Upload功能。

现在Dropbox发布了一个单独的App，可以浏览你保存的所有图片。使用类似时间线的方式呈现，方便你看以前的重要照片。

Carousel puts a lifetime of memories at your fingertips and keeps them safe forever in Dropbox.

Loom 会在5月16日彻底关闭，你可以选择从Loom导出到Carousel，Loom里面的免费空间也会加到的Dropbox空间，我的就加了5G空间。或者下载所有照片作为一个zip文件。

我觉得手机里面最重要的两个东西，一个是联系人，一个是手机拍的照片和视频，虽然我不是一个喜欢拍照片的人，如果手机丢了，但是还有这两个重要资料的话，也不会太伤心，联系人我是使用的Google的备份服务1，照片现在使用的Dropbox的Camera Upload，而如果你安装了Carousel的话，Dropbox会把照片上传工作交给Carousel。

Carousel目前有Android和iPhone版本，还不支持iPad啊。

Excel只选择可视区域

昨天需要加工一个数据，是一些数据的单位有问题，大部分的数据是用的单位吨，而一部分数据使用的是单位吨，所以有1000倍的差距，需要手工修改，是的，使用Excel的选择性粘贴实现，方法是

  1. 筛选需要操作的数据
  2. 在一个空白位置输入1000
  3. 选择需要修改的区域，然后右击选择选择性粘贴
  4. 在选择性粘贴的对话框中选择除

但是，仅仅有一个问题，操作之后呢发现筛选出去的数据也除了1000，我第一次操作没有意识到有这个问题，所以，做出来的数据又全部有问题。这个问题怎么解决呢？方法就是 ALT+;键。 这个快捷键会让Excel只选中可视区域，筛选的区域或者隐藏的区域不会被选择，所以这样操作的时候不会有问题。

Notesuite 上手

昨天晚上买了notesuite，0.99美元，用了之后感觉太糟糕，对中文支持不好，输入的时候不会直接显示文字，所以几乎不能使用中文。

  1. NoteSuite功能相当的多，对待办事项支持应该比Evernote好，因为有单独的待办事项的界面设计
  2. 绘画功能比Notability做的强大，也好用一些。
  3. 界面设计还是有点粗糙。
  4. 有Mac同步app。
  5. 可能是对iPad Mini界面没有做优化，字体，界面按钮都太小。

这个app现在是0.99美元促销，可能在Mac上购买了就可以入手了，其他的话，觉得这个比较糟糕，买Notability比较好。

这个博客怎样建立的

这个博客是使用的Jekyll建立的，放在Github上。因为对git命令不是很熟，所以还是用的SourceTree来提交更新。

具体怎样安装jekyll

因为我是用的OS X，系统自带了Ruby，所以，使用

sudo gem install jekyll

就可以安装jekyll了

jekyll new blog

新建一个文件夹，在blog文件夹中放置你的博客。 在网上下载主题。

jekyll serve

在本地创建和预览你的博客。满意后，将博客传到如github pages上。

编辑博客的话，在Mac上使用的是Mou这个编辑器，在iPad上使用editorial这个编辑器，在手机上使用draft。使用Dropbox同步，在Mac上#使用jekyll build后用source tree更新到GitHub Page上。
